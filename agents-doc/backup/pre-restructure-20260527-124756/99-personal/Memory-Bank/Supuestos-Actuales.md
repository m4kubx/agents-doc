# Supuestos Actuales del Vault y Metodología

Lista viva de supuestos. Actualizar cuando cambien.

- El agente (Kilo u otro) tiene acceso completo de lectura/escritura al vault vía herramientas de filesystem.
- El vault se usa principalmente como memoria de largo plazo + metodología, no como vault personal masivo.
- Se prefiere wikilinks (`[[ ]]`), frontmatter y Bases sobre enlaces markdown normales para mejor integración con Obsidian.
- El Principio de No Generación se respeta estrictamente incluso cuando el agente está mejorando el propio vault.
- Las Skills de Obsidian (json-canvas, obsidian-bases, etc.) se consideran "conocimiento activo" del agente mientras trabaje en este vault.
