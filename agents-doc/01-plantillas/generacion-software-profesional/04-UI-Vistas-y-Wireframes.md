# Sub-Plantilla: UI, Vistas y Flujos de Usuario

Usar durante la **Fase 4** de la plantilla principal.

---

## 1. User Flows Críticos

Identificar y documentar los flujos más importantes (mínimo 6-8):

| # | Flujo | Actor Principal | Pantallas Involucradas | Prioridad |
|---|-------|-----------------|-------------------------|---------|
| 1 | Registro y Onboarding | Usuario nuevo | Landing, Sign Up, Welcome | Alta |
| 2 | Login + Recuperación | Todos | Login, Reset Password | Alta |
| ... | | | | |

Para cada flujo importante, crear un diagrama simple (Mermaid flowchart o secuencia).

---

## 2. Wireframes de Pantallas Principales

Listar las pantallas clave del MVP:

### Pantallas Críticas (MVP)

1. **Home / Dashboard**
   - Objetivo de la pantalla
   - Elementos principales
   - Estados: Loading / Empty / Error / Normal
   - Notas de responsive

2. **Listado de [Entidad]**
   - ...

(Continuar con las 8-12 pantallas más usadas)

**Formato recomendado para cada wireframe:**
- Descripción textual de layout
- Componentes UI identificados
- Interacciones principales
- Datos que muestra

---

## 3. Componentes de UI Reutilizables

Identificar componentes que se repetirán:

- Button variants
- Form controls
- Cards / List items
- Modals / Drawers
- Navigation patterns
- Empty states, error states, loading states

---

## 4. Consideraciones de Diseño

- **Accesibilidad**: Nivel WCAG objetivo
- **Responsive**: Breakpoints principales
- **Idioma y Localización**
- **Dark / Light mode** (si aplica)
- **Animaciones y Micro-interacciones** (solo las importantes)

---

## 5. Herramientas y Entregables

- Wireframes en Markdown + Mermaid (baja fidelidad)
- Referencias a Figma / Excalidraw / Penpot (si existen)
- Design System inicial (tokens, colores, tipografía)
